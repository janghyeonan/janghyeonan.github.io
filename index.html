M사 · WProject 테스트 자동화 환경 구축

- 품질 지표
자동화 커버리지 : 70% 수동 테스트 시간 : 1시간 30분 -> 자동 테스트 30분 수동 테스트 비율 : 50% 감소
- 원인 분석

1. 물류 시스템 중 연동 기능이 많아 단독 기능 테스트 및 사이드 이펙트 리스크가 큼

2. 수동 회귀 테스트 비중이 높아 QA 판단이 경험에 의존
ArgoCD + EKS 배포 구조에 QA 자동화가 포함되지 않음(기존 구축 내용은 단독 사용이라 지속적인 사용이 되지 않았음)

- 테스트 자동화 설계

1. GitHub Actions 기반 CI 트리거 구성
2. API + UI Test(Pytest, Playwright), Language : Python
3. EKS 기반 테스트 실행 환경 분리 운영
4. ArgoCD 배포 전 자동화 테스트 선행 구조 설계

- 개발/구현

1. 테스트 코드와 Workflow 정의를 GitHub에서 관리
2. 커밋/요청 시 GitHub Actions 실행
3. EKS 테스트 환경에서 자동화 실행
4. 자동화 결과를 기반으로 QA 승인 판단
5. 자동화 시스템을 단독으로 설계·구축하고 운영 기준까지 정리

- QA 프로세스 적용

- Before: 수동 테스트 중심, QA 경험 의존, 주관적 안정성 판단
- After: CI 자동화 검증, 데이터 기반 QA 판단, 배포 전 필수 Gate 수행

- 그 외

1. QA 자동화 시스템 단독 구축(타 시스템 영향을 받지 않고 테스트 가능)
2. 자동화 테스트 외 테스트 데이터 및 메서드 생성
3. 슬랙 커맨드 및 알림 기능 구축

C사 · L Project 테스트 자동화 구축

- 품질 지표
자동화 커버리지 80% 수동 테스트 2시간 -> 자동 테스트 30분 수동 테스트 비율 : 70% 감소
- 원인 분석

1. 주간/국가별 배포 구조로 반복적인 회귀 테스트 부담 증가
2. 환경 차이로 자동화 실행 결과 불안정
3. 기존 테스트 자동화는 있었으나 실행 환경 표준화 필요로 개선이 필요하였음

- 테스트 자동화 설계

1. CircleCI 도입
2. Docker로 테스트 환경 이미지화
3. 기존 코드 폐기 후 언어 변경(Python API + UI 자동화)
3. 국가별 서비스 변경 시 탐지 후 그에 맞는 테스트 자동화 실행

- 개발/구현

1. GitHub에 테스트 코드와 Dockerfile 관리
2. CircleCI에서 Docker 컨테이너 실행
3. QA가 필요 시 파이프라인 수동 트리거
4. 웹 + 앱 테스트를 동일 파이프라인에서 실행
5. 자동화 환경을 단독 구축해 재현 가능한 운영 체계 확보

- QA 프로세스 적용

- Before: 로컬 환경 의존, QA 개인 PC 환경 차이 발생
- After: CI + Docker 기반 표준 테스트 환경, 동일 결과 재현

- 그 외

1. FS LMS 자동화 테스트 단독 구축
2. 국가별 상태 코드에 따른 자동화 테스트 구축

N사 · Drone Simulator 자동화 테스트 구축

- 품질 지표
자동화 커버리지 50% 시뮬레이터 수동 테스트 1시간 -> 자동 테스트 20분 수동 테스트 비율 30% 감소
- 원인 분석

1. 웹/앱/시뮬레이터 연동 구조로 수동 테스트 한계
2. 미션 수행 안정성을 수동 검증하기 어려움
3. E2E 테스트 부재

- 테스트 자동화 설계

1. 하드웨어(드론 시뮬레이터) -> Appium(앱) -> Playwright(웹) 자동화 파이프라인
2. 미션 코드를 C++ GTest로 시뮬레이터 단위 테스트 구축
3. GitHub Actions 기반 CI
4. Crontab 정기 자동 실행

- 개발/구현

1. 테스트 서버에서 테스트 코드 pull 후 시뮬레이터 실행
2. 미션 수행 → API 검증 → 앱/웹 결과 확인
3. 시간 기반 자동 실행으로 지속 검증
4. 통합 자동화 환경을 단독 설계·구축하여 E2E 체계 확립

- QA 프로세스 적용

- Before: 물리 드론 테스트 중심, 사전 검증 부족
- After: 시뮬레이터 기반 사전 검증, 비행 안정성 및 비행 전 소프트웨어 점검까지 QA 검증

- 그 외

1. 드론 E2E 자동화 체계 단독 구축
2. 드론 조종기 앱 QA 자동화 테스트 구축
3. 드론 미션 수행 후 데이터 업로드 및 데이터 분석 사이트 자동화 테스트 구축
